Low-cost modern 3D generation

Tiny3D-Latent: a low-level side project with publishable milestones

This plan is for learning modern 3D generation without spending heavily on cloud GPUs, paid APIs, or huge datasets. The project builds a tiny native-3D latent generator from first principles: procedural 3D data, voxel or TSDF representations, a 3D autoencoder, a VAE, a small latent generator, toy text conditioning, mesh extraction, material channels, metrics, and devlogs after every milestone.

Final demo target: text label -> condition vector -> generated latent -> 3D grid -> mesh -> material preview -> report. The goal is not to beat TRELLIS or Hunyuan3D. The goal is to understand the modern pattern at a scale you can build and explain.
1. Synthetic data Generate simple 3D shapes yourself so the project starts cheap and controlled.
2. 3D representation Store shapes as 32^3 occupancy or TSDF grids before exporting meshes.
3. 3D latent model Train a tiny autoencoder and VAE so shapes become compact latent vectors.
4. Latent generation Generate new latents with a small flow or diffusion model.
5. Asset output Decode to a grid, run marching cubes, render previews, and write a report.

What This Project Is

The simple idea

Instead of optimizing a new 3D object from scratch every time, train a small model that learns a reusable 3D shape space. Then generate new shapes inside that learned latent space and decode them into meshes. 

procedural shapes
  -> 3D autoencoder / VAE
  -> latent generator
  -> generated grid
  -> marching cubes mesh
  -> render + report

Why this is modern

Current systems such as TRELLIS, Hunyuan3D, Step1X-3D, and TripoSG use learned 3D representations, latent generation, geometry stages, texture/material stages, and asset export. This project builds a toy-scale version of that direction.

You will not reproduce their scale or quality. You will reproduce the learning path: representation, compression, generation, decoding, mesh extraction, and evaluation.

Cost And Scope Rules

Keep it cheap

  • Use procedural data first.
  • Start at 32^3 grids.
  • Use tiny PyTorch models.
  • Train on CPU, Apple Silicon MPS, or a small local GPU.
  • Avoid paid APIs for the MVP.

Keep it low-level

  • Implement dataset generation.
  • Implement model architecture.
  • Implement training loops.
  • Implement mesh extraction and metrics.
  • Write devlogs from your own experiments.

Do not start with

  • Objaverse-scale datasets.
  • Huge DiT models.
  • Cloud-only workflows.
  • Text-to-image diffusion as a dependency.
  • Benchmarking other models as the main project.

Milestone 1 Data

Procedural 3D Dataset

This milestone creates your own cheap 3D dataset from simple mathematical shapes.

What You Build

  • Occupancy grids for spheres, boxes, rounded boxes, cylinders, capsules, and torus shapes.
  • Random scale, position, rotation, and color labels.
  • Labels like sphere, cube, cylinder, red, blue, tall, and wide.
  • Train and validation splits.
  • A dataset preview script.

Step-By-Step Instructions

  1. Create a normalized coordinate grid from -1 to 1.
  2. Write implicit functions for each shape.
  3. Convert each shape to a binary occupancy grid.
  4. Randomize parameters and save labels.
  5. Render simple previews so you can inspect the data visually.

How It Helps

This avoids large datasets and makes debugging easier. If the model fails on simple shapes you generated yourself, you know the issue is in the pipeline, not in messy real-world data.

Milestone 2 Representation

3D Representation And Mesh Extraction

This milestone proves that the tensor representation your model learns can become an actual mesh asset.

What You Build

  • Dense 32^3 occupancy grid representation.
  • Optional TSDF grid after occupancy works.
  • Slice visualization for inspecting grids.
  • Marching cubes mesh extraction.
  • OBJ or GLB export.
  • Basic mesh stats: vertices, faces, bounds, and file size.

Step-By-Step Instructions

  1. Load one procedural occupancy grid.
  2. Visualize slices through the grid.
  3. Run marching cubes at the occupancy threshold.
  4. Export the resulting vertices and faces as a mesh.
  5. Render or inspect the mesh from multiple views.

How It Helps

Modern 3D generation does not stop at a tensor. This milestone creates the bridge from model output to visible asset. Later, every generated grid will use this same path to become a mesh.

Milestone 3 Compression

Tiny 3D Autoencoder

This milestone trains a small neural network to compress a 3D shape into a latent vector and reconstruct it.

What You Build

  • 3D convolution encoder.
  • Latent vector, for example 64 or 128 dimensions.
  • 3D convolution decoder.
  • Binary occupancy reconstruction loss.
  • IoU reconstruction metric.
  • Original vs reconstructed preview grid.

Step-By-Step Instructions

  1. Start with one shape category and a tiny dataset.
  2. Overfit 16 examples first to prove the network can learn.
  3. Train on the full small dataset after the overfit test passes.
  4. Save reconstructions every epoch.
  5. Track reconstruction loss and IoU.

How It Helps

This is the first core modern idea: a model learns a compressed 3D shape space. Later, the generator will create new points inside this space instead of producing full grids from scratch.

Resources

  • PyTorch Conv3d docs
  • PyTorch BCEWithLogitsLoss docs
  • PyTorch autograd tutorial
  • 3D ShapeNets

Artifact

  -> decoder
  -> reconstructed grid
  -> reconstructed mesh

Done When

  • The model can overfit a tiny batch.
  • Reconstructed shapes are recognizable.
  • The devlog includes loss curves and visual examples.

Milestone 4 Latent Space

Tiny 3D VAE And Latent Interpolation

This milestone makes the latent space smoother and more useful for sampling.

What You Build

  • Mean and log-variance heads.
  • Reparameterization trick.
  • KL loss with warmup.
  • Reconstruction plus KL objective.
  • Latent interpolation between shapes.
  • Random latent samples decoded into meshes.

Step-By-Step Instructions

  1. Convert the autoencoder bottleneck into VAE outputs.
  2. Add KL loss slowly using warmup.
  3. Compare reconstructions before and after KL regularization.
  4. Interpolate between two latent vectors.
  5. Render the interpolation as a grid or animation.

How It Helps

A plain autoencoder can memorize. A VAE encourages the latent space to be smoother, which makes it more realistic to sample and generate new shapes later.

Resources

  • Auto-Encoding Variational Bayes
  • PyTorch VAE example
  • Lilian Weng VAE explanation
  • beta-VAE context optional

Milestone 5 Generation

Latent Generator With Flow Or Diffusion

This milestone generates new 3D shape latents and decodes them into meshes.

What You Build

  • A frozen or reused VAE decoder.
  • A small MLP latent generator.
  • Class conditioning for shape categories.
  • Recommended first version: rectified flow in latent space.
  • Alternative: tiny latent diffusion with an MLP denoiser.
  • Generated samples decoded to grids and meshes.

Step-By-Step Instructions

  1. Encode your training shapes into latent vectors.
  2. Train a small generator from noise plus label to latent.
  3. Decode generated latents with the VAE decoder.
  4. Export meshes and preview them.
  5. Compare generated samples against training examples.

How It Helps

This is the key step where the project becomes generative. You are no longer only reconstructing known shapes. You are creating new shapes from noise and conditions.

Resources

  • DDPM paper
  • Annotated Diffusion
  • Flow Matching
  • Rectified Flow
  • Latent Diffusion

  • Class labels influence the output category.

Milestone 6 Conditioning

Tiny Text Conditioning

This milestone adds a small prompt interface without requiring a large text encoder.

What You Build

  • A small vocabulary parser.
  • Word-to-condition mapping for shape, color, size, and material tokens.
  • Condition vectors injected into the latent generator.
  • Invalid prompt handling.
  • Prompt metadata saved with every generated asset.

Step-By-Step Instructions

  1. Define a small grammar such as <color> <size> <shape>.
  2. Map tokens to one-hot vectors or learned embeddings.
  3. Concatenate the condition vector with noise or timestep features.
  4. Generate several samples per prompt.
  5. Save prompt, tokens, condition vector, and output paths.

How It Helps

This gives you text-to-3D behavior while staying affordable. You learn the core conditioning idea before adding expensive CLIP-style text encoders.

Resources

  • PyTorch Embedding docs
  • CLIP paper for context
  • Classifier-free guidance for context
  • Hugging Face tokenizers course optional

Done When

  • Prompt words change the generated output in visible ways.
  • Unknown prompts fail clearly instead of silently producing nonsense.

Milestone 9 Scaling

Higher Resolution Or Compact Representation

This milestone explores detail, memory cost, and representation tradeoffs.

What You Build

  • A 32^3 vs 64^3 comparison.
  • Optional sparse active voxel storage.
  • Optional TSDF decoder.
  • Optional triplane decoder.
  • Memory, runtime, and quality comparison table.

Step-By-Step Instructions

  1. Measure memory and runtime at 32^3.
  2. Repeat the same experiment at 64^3.
  3. Compare reconstruction quality and mesh detail.
  4. Choose one compact representation only if the dense version is clearly limiting you.
  5. Write down what improved and what became harder.

How It Helps

This is where you learn why modern systems use smarter representations. Dense 3D grids are simple, but memory grows quickly as resolution increases.

Milestone 11 Real Data Extension

Small Real 3D Dataset Ingestion

This milestone teaches the practical mess behind real 3D datasets after the procedural pipeline is already working.

What You Build

  • Download or collect 20 to 100 public meshes.
  • Track source URL and license for every asset.
  • Load mesh formats such as OBJ, GLB, FBX, and STL where possible.
  • Normalize scale so each object fits in the same unit box.
  • Center meshes at the origin.
  • Repair simple mesh issues where possible.
  • Voxelize meshes into 32^3 or 64^3 occupancy/TSDF grids.
  • Create labels such as cup, mug, bottle, or chair.
  • Compare real mesh voxelization against procedural shape voxelization.
  • Test or fine-tune the autoencoder on the small real dataset.

Step-By-Step Instructions

  1. Pick one small category first, such as cups or mugs.
  2. Collect a small number of meshes and record their licenses.
  3. Load each mesh and inspect whether it has missing parts, odd scale, or broken geometry.
  4. Normalize each mesh to a shared coordinate system.
  5. Voxelize the cleaned mesh into the same format used by the procedural dataset.
  6. Generate before/after preview renders and voxel previews.
  7. Run the existing autoencoder or fine-tune it on the real subset.

How It Helps

Real 3D data teaches important production problems: mesh cleanup, scale normalization, voxelization, licensing, inconsistent quality, large downloads, and extra compute. This belongs after the toy pipeline works so dataset complexity does not block the core model.
