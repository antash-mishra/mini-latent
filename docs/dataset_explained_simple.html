Dataset, explained simply

What dataset will we create for Tiny3D-Latent?

We will create a tiny 3D dataset ourselves. It is not a folder of photos. It is not downloaded 3D models. It is a collection of simple 3D objects made with math, saved as small voxel grids, and paired with labels like red sphere or blue tall cylinder.

One-Sentence Explanation

A dataset is just a list of examples the model learns from. In this project, each example is a small 3D grid plus a label that describes the object inside that grid. 

example = {
  label: "red sphere",
  shape_type: "sphere",
  grid: 32 x 32 x 32 occupancy values,
  color: "red"
}

Think Of Pixels, Then Voxels

2D image

A picture is made of pixels

A normal image is a flat grid. Each small square is a pixel. A pixel stores color. 

image size = 32 x 32 pixels
3D object

A voxel object is made of voxels

A voxel grid is like a 3D image. Each tiny cube is a voxel. A voxel stores whether that part of space is filled or empty. 

voxel grid size = 32 x 32 x 32 voxels

What Does The Grid Store?

Occupancy grid

The simplest version stores only 0 or 1.

  • 0 means empty space.
  • 1 means the object fills this voxel.

The blue squares above are a simple 2D slice through a sphere-like 3D grid.

Example: red sphere

For a sphere, voxels near the center are marked 1. Voxels outside the sphere are marked 0. 

if x*x + y*y + z*z <= radius*radius:
    voxel = 1
else:
    voxel = 0

This simple rule creates a 3D ball inside the grid. The label tells the model what the grid represents.

What Shapes Will We Generate?

Shape Why We Use It Example Label
Sphere Simple smooth object, good first test. red sphere
Box / cube Teaches flat faces and sharp edges. blue cube
Rounded box Halfway between cube and sphere. green rounded cube
Cylinder Teaches long objects and circular cross sections. tall yellow cylinder
Capsule Good for smooth elongated shapes. red capsule
Torus Teaches holes and ring shapes. blue torus

How We Build The Dataset

1. Make empty 3D space Create a 32 x 32 x 32 grid. At first, every voxel is empty.
2. Pick a shape Choose sphere, cube, cylinder, capsule, torus, or rounded box.
3. Use a math rule For every voxel, check whether its position is inside the shape.
4. Save the grid Save the voxel grid as a small array file, such as .npy.
5. Save the label Save text like red sphere with the grid so the model knows what it is.

The Build Process In Plain Pseudocode

Simple version

Generate one sphere

grid = empty 32 x 32 x 32

for each voxel in grid:
    get voxel position: x, y, z

    if position is inside sphere:
        grid[x, y, z] = 1
    else:
        grid[x, y, z] = 0

save grid
save label: "red sphere"
Dataset version

Generate many shapes

for i in range(5000):
    shape = random choice:
        sphere, cube, cylinder, capsule, torus

    color = random choice:
        red, blue, green, yellow

    size = random choice:
        small, medium, large, tall, wide

    grid = generate_shape(shape, size)
    label = color + " " + size + " " + shape

    save grid
    save label

Why This Dataset Helps The Model Learn

Shape

It learns categories

The model sees many sphere examples, many cube examples, and many cylinder examples. It starts learning what makes each shape different.

Variation

It learns changes

Because we randomize size and scale, the model learns that a sphere can be small, large, shifted, or stretched.

Conditioning

It learns labels

Labels like red sphere or blue cylinder later let us ask the generator for a specific kind of object.

What The First Milestone Should Produce

The first dataset milestone is done when we can generate many simple shapes and visually confirm that the grids, labels, and previews are correct.

  • At least 1,000 generated examples.
  • A train/validation split.
  • A metadata.json file with labels and shape parameters.
  • A preview image showing many examples.
  • A few meshes extracted from the voxel grids for inspection.

Shortest Summary

We are building a tiny artificial 3D dataset. Each example is a simple object like a sphere, cube, or cylinder. We store it as a 32 x 32 x 32 voxel grid where 1 means filled and 0 means empty. We also store a text label like red sphere. The model learns from many of these examples, then later generates new voxel grids from labels.
